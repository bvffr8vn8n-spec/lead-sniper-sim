############################################

Lead Sniper Simulator

Data Collector (API ФНС)

############################################

Данный проект представляет собой скрипт для автоматизированного сбора базы российских ИТ-компаний с использованием официального API ФНС (api-fns.ru).

Решение ориентировано на практическую аналитическую задачу: данные собираются программно, проходят этап очистки и нормализации и выгружаются в табличный формат, удобный для дальнейшего анализа и работы в Excel.

==================================================

1. Возможности проекта

==================================================

Проект реализует полный цикл работы с данными — от получения сырой информации до формирования готового аналитического датасета.

⸻

1.1 Сбор данных

⸻

В процессе работы используются следующие методы API ФНС:

• /search — первичный поиск компаний
• /multinfo — получение расширенной информации по списку ИНН

⸻

1.2 Извлекаемые данные

⸻

Для каждой компании собираются и нормализуются следующие поля:

– ИНН компании
– наименование компании
– численность сотрудников (числовое значение)
– основной ОКВЭД
– выручка и год выручки (если доступны)
– регион (адрес головного офиса)
– контактная информация

⸻

1.3 Фильтрация

⸻

Реализована фильтрация компаний по следующим критериям:

• численность сотрудников
• ИТ-направление деятельности (ОКВЭД 62, 63, 58)

⸻

1.4 Результат

⸻

Итогом работы является CSV-файл, корректно открывающийся в Excel и готовый для дальнейшего анализа.

==================================================

2. Используемые источники данных

==================================================

⸻

2.1 API ФНС РФ

⸻

В проекте используется официальный API ФНС РФ (api-fns.ru).

Через данный источник получаются следующие сведения:

– регистрационные данные компании
– основной и дополнительные ОКВЭД
– открытые сведения о численности сотрудников (КолРаб)
– финансовые показатели
– адрес и контактная информация

Все данные получаются из легального открытого источника.

3. Структура проекта

==================================================

⸻

3.1 Основные файлы

⸻

src
• main.py — точка входа и управляющая логика
• fetch_list.py — поиск компаний через API
• fetch_multinfo.py — батч-обогащение данных по ИНН
• normalize.py — очистка, нормализация и экспорт данных

⸻

3.2 Выходные данные

⸻

data
• companies.csv — итоговый результат работы скрипта

==================================================

4. Требования и окружение

==================================================

⸻

4.1 Системные требования

⸻

• Python версии 3.10 или выше
• Доступ к API сервиса api-fns.ru

⸻

4.2 Зависимости

⸻

Зависимости устанавливаются стандартным способом через файл requirements.txt.

==================================================

5. Настройка

==================================================

⸻

5.1 API-ключ

⸻

Перед запуском необходимо указать API-ключ сервиса ФНС.

Ключ задаётся через файл .env или через переменные окружения и используется только локально при запуске скрипта.

⸻

5.2 Дополнительные параметры

⸻

При необходимости могут быть настроены:

• количество собираемых компаний
• минимальное количество сотрудников
• поисковый запрос
• задержка между запросами
• максимальное количество страниц поиска

==================================================

6. Запуск проекта

==================================================

⸻

6.1 Запуск скрипта

⸻

Скрипт запускается командой:

python src/main.py

⸻

6.2 Результат выполнения

⸻

После выполнения работы результат сохраняется в файл:

data/companies.csv

Файл приведён к единому формату и готов для анализа в Excel.

==================================================

7. Примечания и ограничения

==================================================

• Численность сотрудников берётся из открытых сведений ФНС
(раздел «Открытые сведения», поле КолРаб)
• Для части компаний отдельные поля могут отсутствовать
— это особенность исходных данных
• Архитектура проекта допускает подключение дополнительных источников
данных без изменения основной логики

==================================================

8. Назначение проекта

==================================================

Проект выполнен в рамках тестового задания и демонстрирует:

• умение работать с внешними API
• навыки программного сбора и очистки данных
• понимание структуры бизнес-данных
• способность создавать воспроизводимые аналитические решения

==================================================
